# x-tweetcraft

**あなたの分身が、あなたの代わりにXツイートを考えてくれるツール。**

AIが書いたと思わせない、「本物のあなたがそこで考えた」かのようなツイート下書きを生成します。

---

## これは何？

「X（Twitter）の投稿、自分で毎回考えるのは大変。でもAIに任せると『AI感』が出てしまって、自分らしくない」

そんな人のために、**あなたの口調・価値観・考え方を深く学習して、あなたの分身となってツイート案を作る** ツールです。

ツールは以下の情報をあなたから学びます：
- あなたの過去のツイート
- あなたのキャリア・原体験・価値観（対話インタビューで掘り下げ）
- あなたの届けたい人・発信したいテーマ

この「分身」は時間とともに賢くなり、使うほど **「本物のあなたが考えたとしか思えない投稿」** を作れるようになっていきます。

### 何ができるの？

- 🎯 **あなたの分身があなたらしい下書きを5〜10個生成**
- 🧠 **インタビューで価値観・原体験を掘り下げ** て、分身に深みを注入
- 📊 **自分の過去投稿を分析** して「何が伸びたか」を学習（API接続後）
- 🔍 **今X上で伸びてる型をリサーチ** してあなたの文脈に翻訳（API接続後）
- 📅 **週次戦略レビュー** で来週の方向性を提案（API接続後）
- 🔄 **使うほど賢くなる** — 採用/却下された下書きから分身が学習（予定）

### 何をしないの？

- ❌ 勝手に投稿はしません（下書きを見せるだけ。投稿はあなたが確認してから）
- ❌ あなたの個人情報をどこにも送りません（すべてあなたのPCで動く）
- ❌ 「AIが書いた感」を出しません（あなたの文体・語尾・視点をそのまま再現します）

## 使うのに必要なもの

このツールは **Claude Code** というツールの上で動きます。

### 1. Claude Code（必須）

Claude Codeは、Anthropic社が作っている「ターミナル上で動くAIアシスタント」です。

- まだ持っていない場合: https://claude.com/claude-code からインストール
- Claude Pro（月額プラン）があると快適に使えます

### 2. X（Twitter）アカウント（必須）

発信したいアカウント。投稿内容の分析やAPI接続に使います。

### 3. X Developer API（任意）

**あると便利だけど、無くても使えます。**

接続すると:
- ✅ 自動投稿できる
- ✅ 過去投稿の深い分析ができる
- ✅ トレンド分析ができる

接続しなくても:
- ✅ ツイート下書きの生成は動く
- ✅ インタビューでbrand voiceを育てられる

## インストール手順

### ステップ1: このリポジトリをダウンロード

ターミナルで、**お好きな場所** にダウンロードできます。どこでもOKです。

**例1: ホームディレクトリに直接置く**
```bash
cd ~
git clone https://github.com/RyoroSuzuki/x-tweetcraft.git
```
→ `~/x-tweetcraft/` に配置されます

**例2: ドキュメントフォルダに置く**
```bash
cd ~/Documents
git clone https://github.com/RyoroSuzuki/x-tweetcraft.git
```
→ `~/Documents/x-tweetcraft/` に配置されます

**例3: 任意のフォルダ名で置く**
```bash
git clone https://github.com/RyoroSuzuki/x-tweetcraft.git ~/my-tools/tweetcraft
```

> **`git` って何？** と思った方: Gitはファイルをダウンロード・管理するツールです。Macなら通常インストール済み。入っていない場合は `xcode-select --install` で入ります。

### ステップ2: Claude Codeでこのプラグインを読み込む

Xを運用したい作業ディレクトリに移動して、**ステップ1でダウンロードした場所** を `--plugin-dir` で指定して起動：

```bash
cd ~/MyWork                               # 自分のX運用用ディレクトリ（お好きな名前でOK）
claude --plugin-dir <ダウンロードしたパス>
```

**実例:**
```bash
# ホームに置いた場合
claude --plugin-dir ~/x-tweetcraft

# Documentsに置いた場合
claude --plugin-dir ~/Documents/x-tweetcraft
```

これで x-tweetcraft の機能が使えるClaude Codeセッションが立ち上がります。

### ステップ3（おすすめ）: 併用すると便利なスキルも入れる

x-tweetcraftと併用すると便利な公式スキルが2つあります：

```bash
claude plugin install document-skills@anthropic-agent-skills
claude plugin install superpowers@claude-plugins-official
```

- **document-skills** → PDFやExcel、Word、PowerPointファイルを読めるようになる（職歴書やブランド資料を自動で取り込める）
- **superpowers** → ブレインストーミングや計画作成の汎用ワークフロー

## 初めて使う（オンボーディング）

Claude Codeを起動して、こう話しかけてください：

```
x-tweetcraftをセットアップして
```

すると以下の流れで進みます：

### ① あなたのXアカウントを教える
URL（例: `https://x.com/YourHandle`）または `@YourHandle` を伝える。

### ② ツイートをいくつか教える
最近の投稿から「これは自分らしい」と思うものを5〜10件コピペ。

（Claudeブラウザ拡張を持っていれば、自動抽出できるプロンプトも提示されます）

### ③ 他の資料があれば渡す
X運用の方針メモ、ブランディング資料、ペルソナ設定などがあればファイルパスかコピペで。無ければスキップOK。

### ④ brand-voice（ブランドボイス）雛形が生成される
あなたのトーン・テーマ・型・NG事項がまとめられたファイルが作られます。

### ⑤ 深掘りインタビューへ進むか選ぶ
「Round 1 だけインタビューしますか？（5分）」と聞かれます。やると投稿の深みが増します。

## 日常の使い方

### 下書きを作る

```
今日のツイート作って
```

→ あなたのbrand-voiceに沿った5〜10個の下書きが、根拠付きで出てきます。

### 気に入ったのを投稿する

API接続済みの場合:
```
3番を投稿して
```

API未接続の場合はクリップボードにコピーされるので、Xに貼り付けて手動投稿。

### フィードバックで分身を育てる（自動）

下書きを選んだ後、こんな感じで伝えてください：

```
3番を採用。ただし文末の「〜だ」を「〜だな」に変えた
7番は好き、でも今日の気分じゃないので次回
2番は違う、上から目線すぎ
```

→ `x-reflect` が自動で learnings.md に記録し、**次回の下書き生成から反映されます**。これが「分身が育つ」仕組みです。使うほどあなたらしくなります。

### ブランドボイスをもっと深める

```
x-interviewで深めたい
```

→ 対話型インタビューが再開。前回の続きから続けられます。

### X APIを接続したくなったら

```
x-connect-apiして
```

→ Developer Portal申請からAPI接続までガイドされます。

### 自分の投稿を分析したい（API接続後）

```
自分の投稿分析して
```

→ 過去投稿の実績・傾向・勝ちパターンをレポート化。

### 今何がウケてるか調べたい（API接続後）

```
今伸びてるツイート調べて
```

→ 市場トレンドを調査して、あなたのbrand-voiceに翻訳した下書き案を提示。

### 週次戦略を考える（API接続後推奨）

```
今週の戦略立てて
```

→ x-content-strategistが起動。先週の振り返り＋トレンド分析＋来週の戦略を提案。

## 自動化：起きたら下書きが出来上がっている体験

「毎朝自分で下書き作るの面倒...」を解決する機能。

### スケジュール設定

```
x-scheduleして
```

または

```
朝までに明日のツイート下書き作っておいて
```

→ x-scheduleスキルが起動して、以下のテンプレートから選べます：

| テンプレート | 内容 | デフォルト時刻 |
|------------|------|---------------|
| **A. 朝の下書き準備** | 自動で市場分析+投稿分析+下書き10案を生成 | 毎朝6時 |
| **B. 週次戦略レビュー** | 先週の振り返り+今週の方向性を提案 | 月曜朝8時 |
| **C. 月次ブランドリフレッシュ** | 過去100件以上を深く分析しbrand-voice更新 | 毎月1日 |
| **D. カスタム** | 独自のスケジュール設定 | 自由 |

### 体験イメージ

```
夜11時にPC放置 → 起床6時 → PC開く
↓
「personal-info/draft-logs/2026-04-06_morning.md」
に今日の下書き10案が既に準備されている
↓
気に入ったのを選んで投稿するだけ
```

### 仕組み

- あなたのPC上で `cron` を使って Claude Code を自動起動
- スケジュール実行時に x-draft などが動く
- 結果は `draft-logs/` や `strategy-reports/` に自動保存
- PCが起動している時だけ動く（PCオフなら次回起動時まで待機）

### 一覧確認・削除

```
スケジュール一覧見せて
```
```
朝の下書き準備を止めて
```

## ブランドボイスってなに？

このツールの中核概念です。あなたの作業ディレクトリに以下が作られます：

```
brand-voice.md                    # 目次とクイックサマリ
personal-info/
├── public-voice.md               # 公的: トーン・テーマ・型・NG事項
├── values-and-origin.md          # 私的: 価値観・原体験・キャリア
├── audience-and-messages.md      # 私的: 届けたい人・コアメッセージ
├── learnings.md                  # x-reflectが蓄積する分身の学習記録
├── schedules.md                  # x-scheduleの登録済み自動化一覧
├── interview-logs/               # インタビュー記録
├── draft-logs/                   # 全下書きの履歴（自動保存）
├── analysis-logs/                # x-analyze-postsのレポート
├── strategy-reports/             # x-content-strategistのレポート
└── trend-reports/                # x-research-trendsのレポート
```

### 2層構造の意図

- **Public（公的レイヤー）** → ツイートに直接使ってOK
- **Private（私的レイヤー）** → 視点や動機の「源」。直接ツイート化しない

なぜ分ける？
- 原体験や挫折話は「公開するまだ準備ができてない」段階のことが多い
- でも、投稿のトーンや視点には滲ませたい
- この分離で「まだ何者でもない段階」でも価値観を保管でき、節目で取り出せる

## PCへの負荷は？

**ゼロです。**

このツールは **Markdown（テキスト）ファイルの集まり** です。Claude Codeが必要な時だけ読むので、バックグラウンドサーバーもプロセスも立ち上がりません。

将来X API接続用の機能を追加しても、Claude Codeセッション中だけ動く軽量プロセスです。

## よくある質問

### Q: 自分の投稿データはどこに送られますか？
A: **どこにも送られません。** 投稿分析はあなたのClaude Code契約のClaude AIとだけやり取りします。Ryoroや第三者のサーバーは経由しません。

### Q: 他のユーザーが使った時、私のClaude契約は消費されますか？
A: **されません。** 各ユーザーは自分のClaude Code契約で動きます。完全に独立しています。

### Q: brand-voiceを他人と共有できますか？
A: 技術的には可能ですが、Privateレイヤーには価値観・原体験など個人情報が入るので、共有する場合はPublicレイヤーのみ抜き出すことを推奨します。

### Q: X以外のSNS（Instagram, Threads, Note等）でも使えますか？
A: 現時点ではX特化ですが、brand-voiceの仕組みは汎用なので将来対応の可能性あり。

### Q: 投稿内容を完全に自動化できますか？
A: 設計上、人間のレビューを挟む想定です。X APIの利用規約も「スパム防止のため人間のレビューを推奨」としています。
